#Тест-кейсы для автотестов на пользовательские сценарии из задания 2.2

##Тест-сьют "Создание задачи"
###Общие предусловия:
1.Открыть браузер Google Chrome версии не ниже 142.
2.Открыть главную страницу сайта по адресу - https://avito-tech-internship-psi.vercel.app/
###Тест-кейс 1: Позитивный e2e тест
###Цель:
Проверить, что задачу можно создать ,вводя валидные данные
###Шаги:
1.Нажать на кнопку "Создать задачу"
2.Ввести название задачи в поле "Название*"
3.Нажать на выпадающий список "Проект*" и нажать на одну из категорий
4.Нажать на выпадающий список "Приоритет*" и нажать на одну из категорий
5.Нажать на выпадающий список "Исполнитель*" и нажать на одну из категорий
6.Нажать на кнопку "Создать"
###Ожидаемый результат:
Задача появилась в списке задач под введеным названием, с выбранными ранее категориями проекта,приоритета,исполнителя
###Фактический результат - соответствует ожидаемому


###Тест-кейс 2:Негативный тест с введением невалидных данных
###Цель: 
Проверить,что задачу невозможно создать,не указав все необходимые параметры
###Шаги:
1.Нажать на кнопку "Создать задачу"
2.Ввести название задачи в поле "Название"
3.Нажать на кнопку "Создать"
###Ожидаемый результат:
Кнопка создать не кликабельна, задача не появляется в списке задач.
###Фактический результат - соответствует ожидаемому

###Шаги:

2.Ввести название длинной более 255 символов задачи в поле "Название*"
3.Нажать на выпадающий список "Проект*" и нажать на одну из категорий
4.Нажать на выпадающий список "Приоритет*" и нажать на одну из категорий
5.Нажать на выпадающий список "Исполнитель*" и нажать на одну из категорий
6.Нажать на кнопку "Создать"
###Ожидаемый результат:
Задача появилась в списке задач под введеным названием, с выбранными ранее категориями проекта,приоритета,исполнителя
###Фактический результат - соответствует ожидаемому

###Тест-кейс 4:Открытие уже созданных задач с листа задач
###Цель: 
Проверить, открываются ли уже созданные задачи.
###Шаги:
1.Нажать на первую в списке задачу
###Ожидаемый результат:
Открывается окно - Редактирование задачи
###Фактический результат - соответствует ожидаемому

###Тест-кейс 5:Редактирование созданных задач
###Цель: 
Проверить, корректно ли изменяются отредактированные данные в созданных задачах
###Шаги:
1. Нажать на первую в списке задачу
2. Открывается окно - Редактирование задачи
3. Вводим новое описание для задачи
4. Вводим новое название для задачи
5. Нажимаем кнопку "Обновить"
###Ожидаемый результат:
На листе задач и при открытии задачи показываются обновленные данные - т.е новое название и описание
###Фактический результат - соответствует ожидаемому
   
###Тест-кейс 6:Осуществление поиска с помощью поисковой строки и выпадающего меню
###Цель: 
Проверить, корректно ли работает поиск задач через поисковую строку и выпадающие списки
###Шаги:
1.В поле вводе поисковой строки ввести название существующей задачи
2.Задача отобразится в листе задач
3.В выпадающем списке "Статус" выбрать один из предложенных статусов задач
4.Выбранный статус должен соответствовать статусу,указанному у задач из списка (напротив названия)
5.В выпадающем списке "Доска" выбрать одну из предложенных досок задач
6.Выбранная доска должна соответствовать доске,указанной у задач из списка (под названием)
###Ожидаемый результат:
На листе задач отображаются только задачи с выбранными параметрами
###Фактический результат:
При выборе статуса "Done" отображаются задачи со всеми тремя статусами(Done/Baklog/In progress)

###Тест-кейс 7:Переход на доски задач с листа задач
###Цель: 
Проверить, возможно ли перейти на страницу с досками со страницы со всеми задачами
###Шаги:
1.На главной странице нажать кнопку "Проекты"
2.Откроется страница с ссылками на доски задач
3.Нажать напротив названия выбранной доски "Перейти к доске"
###Ожидаемый результат:
Откроется страница выбранной доски, на странице будет заголовок- название доски, и все задачи этой доски ,разделенные по статусам.

###Тест-кейс 7:Переход на доски задач с окна - Редактирование задачи
###Цель: 
Проверить, возможно ли перейти на страницу с досками с окна Редактирования задачи
###Шаги:
1.На главной странице нажать на задачу из списка задач
2.Нажать на кнопку "Перейте к доске"
###Ожидаемый результат:
Откроется страница выбранной доски, на странице будет заголовок- название доски, и все задачи этой доски ,разделенные по статусам.
